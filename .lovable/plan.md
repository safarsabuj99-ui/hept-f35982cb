

# Smart Billing and Payment Alert System

## Overview
Add a billing health monitoring system to prevent ad account disablement due to failed payments. This covers new database columns, a notification table, an hourly cron-triggered edge function ("Payment Radar"), a dashboard widget, and simulation updates.

## 1. Database Migration

### New columns on `ad_accounts`
- `billing_type` -- new enum `billing_type` with values `prepaid`, `threshold_postpaid` (default `prepaid`)
- `threshold_limit` -- decimal, nullable, default `250`
- `current_threshold_spend` -- decimal, nullable, default `0`
- `next_billing_date` -- date, nullable
- `card_last_4` -- text, nullable

### New table: `billing_notifications`
Stores alerts generated by the Payment Radar engine.

| Column | Type | Notes |
|---|---|---|
| id | uuid PK | default gen_random_uuid() |
| ad_account_id | uuid FK | references ad_accounts |
| client_id | uuid | owner of the account |
| alert_type | text | `threshold_proximity` or `bill_date_countdown` |
| priority | text | `high`, `medium`, `low` |
| message | text | human-readable alert |
| usage_percent | numeric | current threshold usage % |
| is_read | boolean | default false |
| created_at | timestamptz | default now() |

RLS: admin full access, manager read for managed clients, client read own.

## 2. Edge Function: `billing-radar`

A backend function triggered on a schedule (hourly via cron) or manually from the dashboard.

Logic:
- **Threshold Proximity Alert (80% rule)**: For each active `threshold_postpaid` account, calculate `usage_percent = (current_threshold_spend / threshold_limit) * 100`. If >= 80%, insert a `billing_notifications` row with `alert_type = 'threshold_proximity'` and `priority = 'high'`.
- **Bill Date Countdown (2-day rule)**: For accounts with a `next_billing_date` within 2 days, insert a notification with `alert_type = 'bill_date_countdown'` and `priority = 'medium'`.
- De-duplicates: only creates a new alert if one of the same `alert_type` + `ad_account_id` doesn't already exist for the same day.

Config in `supabase/config.toml`:
```toml
[functions.billing-radar]
verify_jwt = false
```

A `pg_cron` + `pg_net` schedule will call this function every hour.

## 3. Dashboard Widget: `BillingHealthWidget`

New component `src/components/dashboard/BillingHealthWidget.tsx`.

Fetches all active `threshold_postpaid` ad accounts with their billing metadata and unread notifications.

**Visual elements:**
- List sorted by urgency (highest usage % first, then closest billing date)
- Color-coded progress bars per account: green (< 60%), yellow (60-79%), red (>= 80%)
- Display: `[||||||||--] 80% Used ($200/$250)`
- "Due in X days" badges for upcoming billing dates
- Unread notification count badge
- "Mark all read" action button

### Integration into Admin Dashboard

- Icon: `CreditCard` from lucide-react

## Technical Details

### Migration SQL (single migration)
```sql
-- Create billing_type enum
CREATE TYPE billing_type AS ENUM ('prepaid', 'threshold_postpaid');

-- Add columns to ad_accounts
ALTER TABLE ad_accounts
  ADD COLUMN billing_type billing_type NOT NULL DEFAULT 'prepaid',
  ADD COLUMN threshold_limit numeric DEFAULT 250,
  ADD COLUMN current_threshold_spend numeric DEFAULT 0,
  ADD COLUMN next_billing_date date,
  ADD COLUMN card_last_4 text;

-- Create billing_notifications table
CREATE TABLE billing_notifications (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  ad_account_id uuid NOT NULL REFERENCES ad_accounts(id) ON DELETE CASCADE,
  client_id uuid NOT NULL,
  alert_type text NOT NULL,
  priority text NOT NULL DEFAULT 'medium',
  message text NOT NULL,
  usage_percent numeric,
  is_read boolean NOT NULL DEFAULT false,
  created_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE billing_notifications ENABLE ROW LEVEL SECURITY;

CREATE POLICY admin_all_billing_notifications ON billing_notifications
  FOR ALL USING (has_role(auth.uid(), 'admin')) WITH CHECK (has_role(auth.uid(), 'admin'));

CREATE POLICY client_read_own_billing_notifications ON billing_notifications
  FOR SELECT USING (client_id = auth.uid());

CREATE POLICY manager_read_billing_notifications ON billing_notifications
  FOR SELECT USING (
    has_role(auth.uid(), 'manager') AND
    client_id IN (SELECT get_managed_client_ids(auth.uid()))
  );
```
