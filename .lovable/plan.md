# Clear All TikTok Data and Re-Sync Fresh

## What We'll Do

Delete all TikTok-related data from the database tables, then trigger a fresh sync to pull clean data from the TikTok API.

## Data to Clear (in dependency order)


| Table                  | Filter                                                              | Why                                       |
| ---------------------- | ------------------------------------------------------------------- | ----------------------------------------- |
| `daily_metrics`        | WHERE campaign_id IN (campaigns with platform='tiktok')             | Time-series data tied to TikTok campaigns |
| `campaign_performance` | WHERE campaign_id IN (tiktok campaign platform_ids)                 | Legacy performance rows                   |
| `daily_ad_spend`       | WHERE ad_account_id IN (tiktok ad accounts)                         | Daily spend records                       |
| `transactions`         | WHERE description LIKE 'auto_spend:%' AND links to tiktok campaigns | Auto-debit records generated by triggers  |
| &nbsp;                 | &nbsp;                                                              | &nbsp;                                    |
| `campaigns`            | WHERE platform = 'tiktok'                                           | Campaign identity records                 |


Note: `ad_accounts` with `platform_name = 'tiktok'` and `ad_account_clients` junction rows will be **kept** since they define the account structure needed for re-sync. Only the synced data gets wiped.

## Re-Sync

After clearing, trigger `sync-deep-dive` to pull fresh TikTok campaign data, metrics, and statuses from the API. The sync function will recreate campaigns and daily_metrics from scratch.

## Steps

1. Run DELETE statements in correct order (child tables first to avoid FK issues)
2. Trigger `sync-deep-dive` to re-fetch all TikTok data fresh
3. Verify campaigns and metrics are repopulated correctly
